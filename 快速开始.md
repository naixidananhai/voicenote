# 快速开始 - 获取APK并测试

## 🚀 最快方法：从GitHub下载（推荐）

### 步骤1：访问GitHub Actions
打开浏览器访问：
```
https://github.com/naixidananhai/voicenote/actions
```

### 步骤2：找到最新构建
- 查看列表中最新的构建任务（顶部第一个）
- 确认状态是绿色✓（表示构建成功）
- 如果还在构建中（黄色圆圈），等待5-10分钟

### 步骤3：下载APK
- 点击进入构建详情页
- 滚动到页面底部 "Artifacts" 部分
- 点击下载 `app-debug.apk`

### 步骤4：安装到手机
- 将APK传输到手机
- 在手机上安装（可能需要允许"未知来源"）

### 步骤5：测试并反馈
1. 打开应用
2. 授予录音权限
3. 点击"开始监听"
4. 对着手机说话
5. 观察日志面板
6. 点击"📋 复制"按钮复制日志
7. 把日志发给我分析

## 🔧 本地构建（需要完整SDK）

### 当前问题
你的Android SDK不完整，缺少：
- ❌ Android Platform 33 (API 33)
- ❌ Android Platform 34 (API 34)

### 解决方法

#### 方法1：在Android Studio中下载
1. 打开Android Studio
2. 打开项目：`C:\Users\yangyayuan\Desktop\yu`
3. Tools → SDK Manager
4. SDK Platforms标签页：
   - 勾选 "Android 13.0 (Tiramisu)" - API 33
   - 勾选 "Android 14.0 (Upside Down Cake)" - API 34
5. 点击 "Apply" 开始下载
6. 等待下载完成（可能需要10-20分钟）
7. 下载完成后，在项目目录运行：
   ```cmd
   gradlew assembleDebug
   ```

#### 方法2：使用sdkmanager命令行工具
```cmd
cd %LOCALAPPDATA%\Android\Sdk\cmdline-tools\latest\bin
sdkmanager "platforms;android-33"
sdkmanager "platforms;android-34"
```

## 📊 测试重点

安装APK后，重点测试：

1. **VAD检测是否工作**
   - 日志中的"VAD概率"是否大于0
   - 说话时概率是否上升

2. **录音是否触发**
   - 是否看到"🎤 检测到人声，开始录音"
   - 是否看到"🔇 人声结束"

3. **文件是否保存**
   - 点击"打开文件夹"查看文件统计
   - "总录音数"是否增加
